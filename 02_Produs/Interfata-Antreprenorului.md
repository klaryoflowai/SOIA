# Interfata Antreprenorului

Aceasta nota documenteaza cum arata produsul SOIA in forma lui vizibila pentru client.

Sursa de baza este mockup-ul HTML:

- `SOIA_Interface_Mockup_v2.html`

## Cele 2 suprafete principale

### 1. Telegram

Rol:

- Daily Brief matinal
- comenzi rapide
- remindere si follow-up
- control mobil permanent

Elemente vazute in mockup:

- mesaj automat "Daily Brief"
- vanzari de ieri
- oferte active
- progres taskuri echipa
- alerta critica
- comenzi de tip `/status` sau `/trimite raport`
- confirmari automate dupa actiune

### 2. Dashboard web

Rol:

- vizualizare KPI
- alerte si notificari
- taskuri automate vs manuale
- insight-uri AI
- log de automatizari

## Blocurile din dashboard

### KPI cards

- vanzari pe saptamana
- procent taskuri automate
- oferte active
- timp eliberat

### Alerte

Exemple din mockup:

- angajat care nu a trimis raport
- oferta care expira
- factura trimisa automat
- meeting procesat si transformat in actiuni

### Taskuri

Separatie clara intre:

- `AUTO`
- `TU`

Asta este important pentru promisiunea de valoare: clientul vede ce face sistemul si ce ramane in decizia lui.

### AI Advisor

Zona de insight strategic:

- coreleaza campanii si vanzari
- sugereaza urmatorul pas
- transforma datele in recomandari

### Automation log

Arata incredere si trasabilitate:

- ce s-a facut
- cand s-a facut
- pentru cine

## Ce promite produsul prin aceasta interfata

- claritate dimineata
- control in timp real
- separare intre ce face AI si ce decide antreprenorul
- incredere ca sistemul lucreaza si cand antreprenorul nu este prezent

## Cum o prezentam in audit

Nu aratam interfata ca design.

O aratam ca dovada pentru 3 lucruri:

1. "vei sti ce se intampla"
2. "vei putea reactiona rapid"
3. "nu mai urmaresti manual tot"

## Ce trebuie construit ulterior

- versiune demo navigabila
- exemple pe industrii diferite
- set minim de comenzi Telegram standard
- documentatie despre Daily Brief si logica alertelor
